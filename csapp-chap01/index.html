hello, world的生命周期

一个程序的生命周期：

  1. 创建/Create
  2. 编译/Compile
    • 预处理（hello.c → hello.i）
    • 编译（hello.i → hello.s）
    • 汇编（hello.s → hello.o）
    • 链接（hello.o + printf.o → hello）
  3. 运行/Run
  4. 退出/Exit

创建

hello程序保存得到一个后缀名为.c的文件hello.c

编译🎯

接下来，通过gcc -o hello hello.c命令完成对源代码的编译，生成可执行程序hello
。这个过程中编译系统的处理大致可以分为四个阶段，分别为：预处理、编译、汇编以及链接。

预处理

预处理器会根据以#开头的代码，来修改原始程序。例如hello程序中引入了头文件 stdio.h ，预处理器会读取该头文件中的内容，将其中的内容直接插入到源程序中，结果就得到了另外一个 C 程序，这个经过预处理器处理后得到的文件通常以 .i 结尾. hello.c 经过预处理器后得到 hello.i，这个 hello.i 仍旧是一个文本文件。

编译

编译器将 hello.i 文件翻译成 hello.s 文件。这一阶段包括词法分析、语法分析、语义分析、中间代码生成以及优化等等一系列的中间操作。感兴趣的可以了解一下《编译原理》。

汇编

汇编器根据指令集将汇编程序 hello.s 翻译成机器指令，并且把这一系列的机器指令按照固定的规则进行打包，得到可重定位目标文件 hello.o 。此时 hello.o 虽然是一个二进制的文件，但是还不能执行。

链接

在 hello.c 这个程序中，我们调用了printf 函数，printf 函数在是在名为 printf.o 的文件中，这个文件是一个提前编译好的目标文件，链接器(ld）负责把 hello.o 和 printf.o 进行合并。因为链接器要遵循一定规则对 hello.o 和 printf.o 的进行调整，所以 hello.o 才会被称之为可重定位目标文件。最终，hello.o经过链接阶段可以得到可执行目标文件 hello。此时，得到的 hello 就可以被加载到内存中执行了。

系统的硬件组成

总览：

中央处理器（Central Processing Unit, CPU）

中央处理单元/中央处理器。

程序计数器（Program Count, PC）

程序计数器。
PC就是一个4字节（32位机器）或是8字节（64位机器）的存储空间，里面存放的是某一条指令的地址。从系统上电的那一瞬间，直到系统断电，CPU就不断地在执行PC指向的指令，然后更新PC，使其指向下一条要执行的指令。

寄存器文件（Register File）

寄存器文件。
它是 CPU 内部的一个存储设备。通俗的讲，寄存器可以理解为一个临时存放数据的空间。
例如，计算两个变量 a+b 的和，处理器从内存中读取 a 的值暂存在寄存器 X 中，读取 B 的值暂存在寄存器 Y 中，这个操作会覆盖寄存器中原来的数值。

算术逻辑单元（Arithmatic/logic Unit，ALU）

算术逻辑单元。
处理器完成加载的操作后，ALU（Arithmatic/logic Unit）会从复制寄存器 X 和 Y 中保存的数值，然后进行算术运算，得到的结果会保存到寄存器 X 或者寄存器 Y 中，此时寄存器中原来的数值会被新的数值覆盖。

内存（Main Memory）

内存，也称为主存。处理器在执行程序时，内存主要存放程序指令以及数据。
从物理上讲，内存是由随机动态存储器芯片组成；从逻辑上讲，内存可以看成一个从零开始的大数组，每个字节都有相应地址。

总线（Bus）

内存和处理器（CPU）之间通过总线（Bus）来进行数据传递。实际上，总线贯穿了整个计算机系统，它负责将信息从一个部件传递到另外一个部件。
通常总线被设计成传送固定长度的字节块，也就是字（word）。至于这个字到底是多少个字节，各个系统中是不一样的，32位的机器，一个字长是4个字节；而64位的机器，一个字长是8个字节。

输入输出设备（I/O）

除了处理器、内存以及总线，计算机系统还包含了各种输入输出设备，例如键盘、鼠标、显示器以及磁盘等等。
每一个输入输出设备都通过一个控制器（Controller）或者适配器（Adapter）与 IO 总线相连。

程序是怎样跑起来的

Step1

首先,我们通过键盘输入”./hello” 的字符串，shell 程序会将输入的字符逐一读入寄存器，处理器会把 hello 这个字符串放入内存中。

Step2

当我们完成输入，按下回车键时，shell 程序就知道我们已经完成了命令的输入，然后执行一系列的指令来来加载可执行文件 hello ，这些指令将 hello 中的数据和代码从磁盘复制到内存。
数据就是我们要显示输出的 “hello , world\n” ;当可执行文件 hello 中的代码和数据被加载到内存中，处理器就开始执行 main 函数中的代码。

Step3

CPU会将 “hello , world\n” 这个字符串从内存复制到寄存器文件。然后再从寄存器文件复制到显示设备，最终hello , world显示在屏幕上。

存储设备

存储设备的容量

寄存器文件（Register File）: 100-1000B
高速缓存 L1 cache：10-100KB
高速缓存 L2 cache：0.1-10MB
高速缓存 L3 cache：10-100MB
内存（Main Memory）：1-100GB
磁盘（Disk）：1-1000TB

存储设备的层级结构

从这个层次结构来看，从上到下，设备的访问速度越来越慢，容量越来越大，每字节的造价也越来越便宜。这个层次结构的主要思想就是：上一层存储设备是下一层存储设备的高速缓存。

超线程（Hyperthreading）

当一个线程因为读取数据而进人等待状态时，CPU可以去执行另外一个线程，其中线程之间的切换只需要极少的时间代价。

指令级并行（Instruction-level Parallelism）

现代处理器可以同时执行多条指令的属性称为指佘级并行，在第4章中将介绍流水线技术。

单指令多数据并行

现代处理器拥有特殊的硬件部件，允许一条指令产生多个并行的操作，这种方式称为单指令多数据（Single-Instruction Multiple-Data Parallelism）。
